<task id="02-01-T1">
<name>Extend schema with course + ingestion tables</name>
<read_first>
- src/lib/db/schema.ts (current auth-only schema to extend)
</read_first>
<action>
Add these four tables to src/lib/db/schema.ts after the existing auth tables:

courses: id (uuid pk), creator_id (text, FK users.id cascade), title (text notNull), description (text), topic (text notNull), length_preset (text notNull — "quick"|"standard"|"long"), visibility (text notNull default "private" — "public"|"private"), status (text notNull default "generating" — "generating"|"ready"|"error"), created_at (timestamp defaultNow notNull)

lessons: id (uuid pk), course_id (text notNull FK courses.id cascade), position (integer notNull), topic (text notNull), youtube_video_id (text), video_title (text), video_duration_seconds (integer), transcript_cached (text), transcript_status (text notNull default "pending" — "pending"|"fetched"|"unavailable"), created_at (timestamp defaultNow notNull)

questions: id (uuid pk), lesson_id (text notNull FK lessons.id cascade), position (integer notNull), question_text (text notNull), options (text notNull — JSON array of 4 strings), correct_index (integer notNull), created_at (timestamp defaultNow notNull)

user_progress: id (uuid pk), user_id (text notNull FK users.id cascade), course_id (text notNull FK courses.id cascade), lesson_id (text notNull FK lessons.id cascade), passed_quiz (boolean notNull default false), completed_at (timestamp), created_at (timestamp defaultNow notNull). Add unique constraint on (user_id, lesson_id).
</action>
<acceptance_criteria>

- user_progress has a unique compound key on (user_id, lesson_id)
</acceptance_criteria>
