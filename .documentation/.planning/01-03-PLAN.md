  <action>
    Create `src/actions/password-reset.ts` with `'use server'`. Imports:
    ```typescript
    'use server'
    import { db } from "@/lib/db"
    import { users, passwordResetTokens } from "@/lib/db/schema"
    import { sendPasswordResetEmail } from "@/lib/mail"
    import bcrypt from "bcryptjs"
    import { randomBytes } from "crypto"
    import { eq, and, gt } from "drizzle-orm"
    import { redirect } from "next/navigation"
    ```

    `requestPasswordReset(state, formData)` (RESEARCH §Pattern 4 Step 1):
    1. Read and trim `email` from formData. (Optionally validate with a Zod email check.)
    2. Look up user: `const user = await db.query.users.findFirst({ where: eq(users.email, email) })`.
    3. SECURITY: Do NOT leak whether the email exists. Always return the same success-style message: `{ message: "Check your inbox — we sent a reset link to " + email + "." }`. Only actually generate+send if the user exists.
    4. If user exists: delete any existing token for this email (`db.delete(passwordResetTokens).where(eq(passwordResetTokens.email, email))`), generate `const token = randomBytes(32).toString("hex")`, insert with 1-hour expiry (`expires: new Date(Date.now() + 60*60*1000)`), then `await sendPasswordResetEmail(email, token)`.

    `applyPasswordReset(state, formData)` (RESEARCH §Pattern 4 Step 2):
    1. Read `token` and `password` from formData. Validate password min 8 chars; on failure return `{ errors: { password: ["Password must be at least 8 characters."] } }`.
    2. Find a non-expired token: `const row = await db.query.passwordResetTokens.findFirst({ where: and(eq(passwordResetTokens.token, token), gt(passwordResetTokens.expires, new Date())) })`. If none, return `{ message: "This reset link is invalid or has expired. Request a new one." }`.
    3. Hash: `const hashed = await bcrypt.hash(password, 12)`.
    4. `await db.update(users).set({ password: hashed }).where(eq(users.email, row.email))`.
    5. Delete the token (single-use): `await db.delete(passwordResetTokens).where(eq(passwordResetTokens.token, token))`.
    6. `redirect("/login")` (let it propagate — do not catch).

    Use the `FormState` type from `@/lib/definitions` for the state params, or define a local extended type if a password-only error field is needed.
  </action>
